# 1. Cargar los datos
df = pd.read_csv('../../train.csv')

# 2. Crear la variable de respuesta categórica basada en los terciles del precio de venta
tercil_1 = df['SalePrice'].quantile(1/3)
tercil_2 = df['SalePrice'].quantile(2/3)

# Crear la categoría de precio
df['PriceCategory'] = pd.cut(
    df['SalePrice'], 
    bins=[0, tercil_1, tercil_2, float('inf')], 
    labels=['Económica', 'Intermedia', 'Cara']
)

# 3. Seleccionar las características para el modelo
selected_features = [
    'OverallQual',    # Calidad general
    'GrLivArea',      # Área habitable
    'GarageCars',     # Capacidad del garaje
    'YearBuilt',      # Año de construcción
    'TotalBsmtSF',    # Superficie total del sótano
    'FullBath',       # Baños completos
    'Neighborhood',   # Vecindario (categórica)
    'ExterQual',      # Calidad del exterior (categórica)
    'KitchenQual',    # Calidad de la cocina (categórica)
    'BsmtQual'        # Calidad del sótano (categórica)
]

# 4. Preparar los datos
X = df[selected_features].copy()
y = df['PriceCategory']

In [68]:
# Manejar valores faltantes en características numéricas
numeric_features = X.select_dtypes(include=['int64', 'float64']).columns
imputer = SimpleImputer(strategy='median')
X[numeric_features] = imputer.fit_transform(X[numeric_features])

# Manejar valores faltantes en características categóricas
categorical_features = X.select_dtypes(include=['object']).columns
X[categorical_features] = X[categorical_features].fillna('None')

# Objetos de preprocesamiento para pipeline de modelos
preprocesador_categorico = OneHotEncoder(handle_unknown="ignore")
preprocesador_numerico = StandardScaler()

# Dividir los datos en conjunto de entrenamiento y prueba
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42, stratify=y
)

Este modelo utiliza una arquitectura de dos capas ocultas con 5 y 3 neuronas respectivamente, y la función de activación tanh, que produce salidas entre -1 y 1. Conclusión entre ellos¶

El Modelo 1, que utiliza la función de activación tanh y una arquitectura más compacta de dos capas ocultas (5, 3), logra un desempeño general sólido, con un accuracy del 80.36% y un f1-score promedio ponderado también cercano a 0.80. Su matriz de confusión muestra un buen rendimiento en la categoría "Cara", pero se nota cierta dificultad en clasificar correctamente las casas "Intermedias", con una recall de solo 0.69 para esa clase. Además, su tiempo de procesamiento fue ligeramente más alto, de aproximadamente 0.90 segundos.

Por otro lado, el Modelo 2, con una única capa oculta de 10 neuronas y función de activación relu, ofrece un rendimiento ligeramente superior en casi todos los aspectos. Su accuracy alcanza el 80.82%, y mejora levemente el f1-score global, especialmente en la clase "Económica". Aunque la mejora no es drástica, sí es consistente, y además lo logra en un tiempo de procesamiento menor (0.69 segundos), lo que lo hace más eficiente computacionalmente.

Aunque ambos modelos muestran un rendimiento similar, el Modelo 2 es la mejor opción en este caso. No solo obtiene mejores métricas globales y por clase, sino que también lo hace de forma más eficiente. Su arquitectura más simple y moderna (una sola capa con relu) parece adaptarse mejor a la estructura del conjunto de datos sin comprometer la generalización. Esto lo convierte en una base más sólida para continuar con ajustes o pruebas adicionales.

Al comparar el modelo original con el modelo tuneado mediante GridSearchCV, se observa que el rendimiento general no mejoró significativamente. De hecho, el modelo original presentó una precisión ligeramente superior (80.8% frente a 80.6%), así como mejores métricas en la clase “Intermedia”, la más difícil de predecir. Además, la curva de pérdida del modelo tuneado (en verde) muestra una trayectoria más irregular en las primeras iteraciones, lo que podría indicar cierta inestabilidad en la convergencia o una mayor sensibilidad a los parámetros elegidos. En cuanto a la matriz de confusión, los resultados son muy similares entre ambos modelos, sin diferencias significativas en las clasificaciones correctas o errores. Por lo tanto, aunque el proceso de ajuste no perjudicó gravemente el desempeño, tampoco aportó mejoras claras. Esto sugiere que el modelo se encuentra en una etapa en la que afinarlo más podría tener beneficios marginales. Sin embargo, todavía es posible explorar otras estrategias de mejora como el ajuste más fino del learning_rate, usar early stopping más agresivo, probar otras funciones de activación o técnicas de regularización, siempre validando con métodos robustos para evitar el sobreajuste.
